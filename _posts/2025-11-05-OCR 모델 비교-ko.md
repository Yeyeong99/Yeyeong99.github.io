# PDF 스캔에 사용 가능한 OCR

|| 업스테이지 OCR | Docling |
|---|---|---|
|소개|• 업스테이지에서 구축한 OCR모델</br• 텍스트 + 위치 정보 + 구조화 된 데이터 변환 </br>• OCR + 문서 파싱, 키-밸류 추출, 문서 구조 분석 포함|• OCR 파이프라인</br>• 기존의 OCR을 활용해 레이아웃을 인식하고 내보내는 기능 지원(통합 툴체인)|
|언어 특징|한국어 인식에 강함, 한국어 영어 특수문자 숫자 한자 등 지원|Rapid OCR 사용 시 중국어가 디폴트, 사용하는 OCR 엔진에 따라 달라짐|
|구조 인식|• 표 구조 유지를 위해선 별도 설정 필요</br>• 바운딩 박스(단어 위치) 반환|• 표 구조 인식 가능</br>• RT-DETR + DocLayNet 기반 모델로 페이지를 구획</br>• 문서 전체 레이아웃, 이미지, 리스트 유지, 마크다운으로 반환|


# 실제 사용해봤을 때
- 업스테이지 OCR과 Rapid OCR(Docling)의 한국어 인식 차이는 크게 느끼지 못했음
  - 다만 특정 문서 형식에서 한국어 인식을 아예 못하고 한자로 반환하는 경우가 있었음
  - 
## 업스테이지
- 대부분의 경우 성능이 준수하지만 숫자를 제대로 인식하지 못하는 경우가 있었음

## Docling
- 레이아웃 유지가 장점이라고 했는데, 별도의 열이 하나로 합쳐지거나, 표가 두 페이지에 걸쳐 있을 경우 하나로 인식하지 못함
- 문서 파싱에서 소제목의 순서가 바뀌거나 하는 식으로 문서 상의 순서가 제대로 유지 안되는 경우가 종종 있었음
- 표를 이미지로 처리해버리는 경우가 있었음
  
### 가능한 원인
- 표 외곽선: 약하거나 인식하기 힘든 경우
- 주된 내용이 숫자: 셀의 좌우 정렬 패턴이 일정하지 않을 가능성이 높음 => 하나의 열로 인식하기 어려움
- pdf DPI가 낮은 경우
